# 游戏产品经理分析框架
# 来源：产品方法论 / 竞品分析实践 / 腾讯游戏学堂方法整理

## 竞品分析框架

### 五维评估模型
分析任何竞品时，沿以下五个维度展开：
1. 核心玩法：乐趣来源是什么，可重玩性如何
2. 变现策略：靠什么赚钱，付费深度如何
3. 用户体验：流畅度、视觉、音效，沉浸感如何
4. 留存机制：每日任务、内容更新、社交功能
5. 用户口碑：评分分布、差评主题、正面情感

### 差异化机会识别
当多款竞品都有同一个高频投诉时 → 这是市场空白
当某款竞品有独特正面评价时 → 这是可借鉴的成功要素
当所有竞品都没有做的功能时 → 这是创新机会点

## 产品决策框架

### 优先级评估公式
优先级分数 = （用户影响范围 × 留存/收入影响 × 数据可信度）÷ 实现成本

各因子说明：
- 用户影响范围：受影响评论占比（0-1）
- 留存/收入影响：对核心KPI的预期影响（0-1）
- 数据可信度：样本量和证据质量（0-1）
- 实现成本：工程和产品资源投入（0-1，越高越贵）

### P0/P1/P2分级标准
- P0（本迭代必做）：影响超过30%用户 + 直接影响留存或收入
- P1（下迭代计划）：影响10-30%用户 + 中等业务影响
- P2（排队候选）：影响少于10%用户 + 体验优化类

## App Store评论分析方法论

### 评论样本量要求
- 基础分析：至少200条评论
- 可信结论：至少500条评论
- 跨市场对比：每个市场至少100条

### 情感分析解读规则
- VADER分数 > 0.05：正面情感
- VADER分数 < -0.05：负面情感
- -0.05 到 0.05：中性
- 整体均分低于-0.1：产品存在严重体验问题
- 广告主题均分低于-0.3：广告策略需要立即调整

### 评论时间维度分析
- 近30天评论 vs 历史评论：追踪版本更新影响
- 评分趋势下滑：通常对应某次版本更新
- 特定日期差评集中：排查该时期的版本变更

## 数据驱动决策流程

### 标准分析流程
1. 定义问题：用户反馈中最高频的痛点是什么
2. 量化规模：影响多少用户，占比多少
3. 挖掘原因：这个痛点背后的根本原因是什么
4. 设计方案：至少给出3个不同成本的解决方案
5. 评估影响：预期改善哪个指标，改善幅度是多少
6. 制定实验：如何验证方案有效（AB测试设计）

### AB测试设计原则
- 最小样本量：每组至少500用户，运行至少7天
- 主指标：一次实验只看一个核心指标
- 护栏指标：确保实验组的留存/收入不低于对照组
- 结论标准：p值<0.05才算统计显著

## 产品陈述结构（面试适用）

### STAR+数据结构
- Situation：背景（用户痛点/市场机会）
- Task：目标（要解决什么问题）
- Action：行动（做了什么，为什么这样做）
- Result：结果（数据证明效果）
- 数据支撑：每个环节都要有具体数字

### 常见面试追问和回答框架
问：「数据从哪来的？」
答：说明数据来源（用户评论/AB测试/行业报告）和样本量

问：「为什么不做XXX方案？」
答：说明方案对比的trade-off，你的决策依据

问：「如果数据不支持你的结论怎么办？」
答：说明你的假设和验证方式，以及如何快速迭代
