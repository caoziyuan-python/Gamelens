# UI 规格文档

## 文档1：用户流程文档

步骤1：首次进入应用 → 系统显示侧边栏主标题 `GameLens 🎮`、副标题“海外手游竞品洞察助手 — 让用户评论变成产品决策”，默认展示游戏选择、分析模式、竞品搜索和高级设置区域。  
步骤2：用户在侧边栏查看默认游戏列表 → 系统通过下拉框展示内置游戏与已添加竞品。  
步骤3：用户在“选择游戏”下拉框中选择目标游戏 → 系统记录当前目标游戏，等待用户开始分析。  
步骤4：用户在“分析模式”单选框中选择“完整模式（含AI）”或“快速模式（规则引擎）” → 系统切换后续分析时是否启用 LLM。  
步骤5：用户点击“开始分析” → 系统优先检查会话缓存和磁盘缓存；若未命中，则抓取评论、执行分析流水线、保存结果，并更新当前游戏状态。  
步骤6：评论抓取完成后 → 系统执行情感分析、规则主题分类、LLM 主题归纳/建议生成（若启用 AI），同时自动构建该游戏的 RAG 索引。  
步骤7：分析完成后 → 系统展示三个主 tab：`📊 竞品雷达`、`💬 用户声音`、`🎯 产品决策`。  
步骤8：用户进入“📊 竞品雷达” → 系统展示核心结论、评论总数、正向情感占比、模型一致性、核心洞察卡片、跨游戏对比表和竞品机会点。  
步骤9：用户在“📊 竞品雷达”查看多游戏数据 → 系统按当前缓存游戏展示评论数、平均评分、正向情感占比和核心问题；若游戏不足两款，则提示先分析更多游戏。  
步骤10：用户进入“💬 用户声音” → 系统展示评分分布图、情绪占比图、评分-情绪关联图、主题热力图、主题列表和原始评论表。  
步骤11：用户进入“🎯 产品决策” → 系统展示 P0/P1/P2 产品建议卡片，每张卡片包含建议内容、反馈按钮、PRD 导出、复制到飞书和证据展开区。  
步骤12：用户点击某条建议卡片的“有帮助”或“没帮助” → 系统记录反馈数据，并提示“反馈已保存”。  
步骤13：用户点击某条建议卡片的“导出 PRD” → 系统下载该建议对应的 Markdown PRD 文件。  
步骤14：用户点击某条建议卡片的“复制到飞书” → 系统展开 Markdown 内容，并提示可直接复制到文档工具中。  
步骤15：用户点击某条建议卡片的“查看证据” → 系统展开展示与该洞察相关的代表性评论。  
步骤16：用户在“🎯 产品决策”中的 AI 助手输入框输入追问 → 系统调用 `run_agent`，基于当前分析结果生成中文回答并显示在对话区域。  
步骤17：用户在“🔍 深度竞品分析”输入问题 → 系统等待用户选择一个已分析且已建立索引的竞品。  
步骤18：若当前没有可用于深度对比的竞品索引 → 系统提示“请先分析一款竞品游戏，即可启用深度竞品分析功能”。  
步骤19：用户选择“对比竞品”并点击深度分析区的“开始分析” → 系统通过 `st.status()` 分四步展示 Agent 执行过程：提取关键词、检索当前游戏评论、检索竞品评论、生成差距报告。  
步骤20：Deep Dive Agent 执行完成 → 系统展示 `gap_summary`、`root_causes` 列表和 `priority_score`；若失败则展示 warning 提示。  
步骤21：用户在侧边栏“竞品搜索”输入关键词并点击“搜索竞品” → 系统调用商店搜索接口，返回候选竞品列表。  
步骤22：用户点击某个搜索结果的“添加” → 系统把该竞品加入游戏列表，并允许后续分析与对比。  
步骤23：用户展开侧边栏“⚙️ 高级设置” → 系统展示 Level 2 验证、重新分析、刷新评论、批量分析、缓存加载、学习反馈和知识库管理等高级功能。  
步骤24：用户点击“重新分析” → 系统保留评论缓存，仅重新执行分析流程。  
步骤25：用户点击“刷新评论” → 系统重新抓取评论并重新分析。  
步骤26：用户点击“批量分析全部游戏” → 系统对当前游戏池逐个抓取、分析并缓存结果，完成后可用于跨游戏对比。  
步骤27：用户点击“加载”某条缓存分析结果 → 系统把缓存结果恢复为当前分析视图。  
步骤28：用户点击“重建知识库索引” → 系统重新读取 `knowledge/` 并构建通用知识索引。  
步骤29：用户点击“生成评论知识库” → 系统将当前已分析游戏的评论摘要加入知识库并重建索引。
