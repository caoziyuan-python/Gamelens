# Prompt汇总使用说明

这个文件夹用于长期维护“可直接复用”的 prompt。

## 文件命名规则

统一使用：

`主功能-关键限制-适用对象.md`

例如：

- `代码注释-只加注释-单文件.md`
- `需求拆解-结构化输出-产品经理版.md`
- `竞品分析-表格输出-通用版.md`
- `SQL生成-自然语言转SQL-分析场景.md`

命名原则：

1. 不加日期，优先保证调用方便
2. 文件名只表达“这是干什么的”
3. 最多保留 3 段，避免过长
4. 优先写“功能词 + 限制词 + 对象词”

不推荐：

- `prompt1.md`
- `最终版.md`
- `很好用的注释prompt.md`
- `2026-04-06-代码注释.md`

## 三类关键词

功能词：

- `代码注释`
- `需求拆解`
- `竞品分析`
- `SQL生成`
- `PRD撰写`

限制词：

- `只加注释`
- `不改逻辑`
- `结构化输出`
- `口语化`
- `产品经理版`

对象词：

- `单文件`
- `接口文档`
- `用户评论`
- `数据表`
- `会议纪要`

## 文件内容规则

每个 prompt 文件本身就是最终可用版，不再拆成“通用版”和“项目版”两份。

默认规则：

- 没有打标的内容都视为可复用
- 只有换项目后必须修改的内容才打标
- 统一使用 `【需按项目替换：...】`

统一标记示例：

- `【需按项目替换：项目名称】`
- `【需按项目替换：文件名】`
- `【需按项目替换：项目背景】`
- `【需按项目替换：字段名】`
- `【需按项目替换：业务目标】`
- `【需按项目替换：粘贴代码】`

## 推荐文件结构

```md
# 标题

## 用途
一句话说明这份 prompt 用来解决什么问题

## 使用说明
- 只修改带有 `【需按项目替换】` 标记的内容
- 其余部分默认直接复用

## Prompt正文
[这里放完整 prompt]

## 备注
- 可选：适用范围
- 可选：补充说明
```

## 维护建议

1. 新增 prompt 时先判断是否能复用已有文件
2. 如果只是小改动，优先更新原文件，而不是新建“最终版2”
3. 新增后同步更新 `目录索引.md`
4. 如需新建模板，优先复用 `Prompt模板-项目内容打标.md`
