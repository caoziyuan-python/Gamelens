# Figma 设计简报

## 第一部分：设计目标

GameLens 是一个面向海外手游产品经理的竞品情报分析平台，核心目标是把 App Store 评论转化为可执行的产品判断。目标用户是需要快速理解竞品优势、用户痛点和功能优先级的游戏 PM、策略分析和产品面试展示场景使用者。核心使用场景包括：选择一款游戏并快速完成评论分析、横向查看多款竞品差异、从用户原声中提炼产品建议，以及通过 Deep Dive Agent 对某个具体问题进行可观察的深度竞品分析。

## 第二部分：完整用户路径

### Step 1: 首次进入
用户动作：
- 打开应用
- 观察页面整体布局

界面变化：
- 左侧展示侧边栏，顶部包含产品标题和副标题
- 主区域默认处于未分析状态，提示先选择游戏并开始分析

关键 UI 元素：
- 侧边栏标题 `GameLens 🎮`
- 副标题 `竞品情报分析平台`
- 游戏选择下拉框
- 分析模式单选框
- 开始分析按钮

### Step 2: 选择游戏
用户动作：
- 在“选择游戏”下拉框中选择一款目标游戏
- 根据需要切换“完整模式（含AI）”或“快速模式（规则引擎）”

界面变化：
- 当前分析目标被切换
- 分析模式决定后续是否启用 LLM

关键 UI 元素：
- 游戏选择下拉框
- 分析模式单选框
- 开始分析按钮

### Step 3: 开始分析
用户动作：
- 点击“开始分析”

界面变化：
- 系统优先检查会话缓存和磁盘缓存
- 如无缓存则抓取评论并执行分析流水线
- 分析完成后主界面切换到三大 tab 视图

关键 UI 元素：
- 开始分析按钮
- 加载中状态提示
- 成功 / 提示信息条

### Step 4: 查看竞品雷达
用户动作：
- 进入 `📊 竞品雷达` tab
- 查看总览指标和核心洞察
- 浏览跨游戏对比表和竞品机会点

界面变化：
- 显示核心结论区
- 显示三张指标卡：评论总数、正向情感占比、模型一致性
- 显示 P0 / P1 / P2 洞察卡片
- 显示跨游戏对比表
- 显示差异化优势、共性问题、机会空白

关键 UI 元素：
- Tab 导航
- 指标卡
- 洞察卡片
- 数据表格
- 信息提示区

### Step 5: 查看用户声音
用户动作：
- 进入 `💬 用户声音` tab
- 浏览情绪、主题和评论原文

界面变化：
- 显示四类分析图表
- 显示用户主题列表
- 显示原始评论表格

关键 UI 元素：
- 评分分布图
- 情感分布图
- 评分-情感关联图
- 主题热力图
- 用户主题列表
- 原始评论表格

### Step 6: 查看产品决策
用户动作：
- 进入 `🎯 产品决策` tab
- 浏览建议卡片
- 对建议做反馈或导出

界面变化：
- 显示多张 P0 / P1 / P2 建议卡片
- 每张卡片下方展示交互按钮
- 可展开查看证据评论

关键 UI 元素：
- 建议卡片
- 有帮助 / 没帮助按钮
- 导出 PRD 按钮
- 复制到飞书按钮
- 查看证据 Expander

### Step 7: 使用 AI 助手
用户动作：
- 在 AI 助手输入框中输入追问

界面变化：
- 系统基于当前分析结果生成中文回答
- 对话以聊天形式展示

关键 UI 元素：
- AI 助手输入框
- 用户消息气泡
- 助手消息气泡
- 加载中状态

### Step 8: 使用 Deep Dive Agent
用户动作：
- 在“深度竞品分析”区域输入自然语言问题
- 选择一个已分析竞品
- 点击“开始分析”

界面变化：
- 展开四步状态面板
- 顺序展示关键词提取、目标游戏检索、竞品检索、差距分析
- 完成后显示差距总结、核心原因和优先级评分

关键 UI 元素：
- 深度竞品分析标题和说明
- 问题输入框
- 对比竞品下拉框
- 开始分析按钮
- 四步执行状态面板
- gap_summary 结果提示
- root_causes 列表
- priority_score 指标卡

### Step 9: 搜索竞品
用户动作：
- 在侧边栏“竞品搜索”输入关键词
- 点击“搜索竞品”
- 在结果列表中点击“添加”

界面变化：
- 列表显示商店搜索结果
- 点击“添加”后，竞品会进入游戏选择池

关键 UI 元素：
- 竞品搜索输入框
- 搜索竞品按钮
- 搜索结果列表
- 添加按钮

### Step 10: 使用高级设置
用户动作：
- 展开 `⚙️ 高级设置`
- 使用重新分析、刷新评论、批量分析、缓存加载或知识库管理功能

界面变化：
- 高级功能区展开
- 可执行批量分析、加载历史缓存、查看学习反馈、管理知识库索引

关键 UI 元素：
- 高级设置 Expander
- 启用 Level 2 验证 checkbox
- 重新分析按钮
- 刷新评论按钮
- 批量分析全部游戏按钮
- 批量启用AI checkbox
- 加载缓存按钮
- 重建知识库索引按钮
- 生成评论知识库按钮

## 第五部分：色彩与字体规范建议

### 色彩系统建议

| 用途 | 建议颜色 | 说明 |
| --- | --- | --- |
| 页面主背景 | `#0D0F14` | 全局深色背景 |
| 侧边栏背景 | `#111318` | 与主内容区形成层级区分 |
| 卡片背景 | `#161920` | 指标卡、洞察卡、状态面板统一容器色 |
| 边框色 | `#1E2128` | 弱对比边框 |
| 次级边框 / 输入框边框 | `#2A2D3A` | 输入与按钮 hover 边框 |
| 主品牌色 | `#00C896` | 主按钮、激活 tab、重点高亮 |
| 主品牌色 Hover | `#00DBA6` | 主按钮 hover |
| 主文字 | `#FFFFFF` | 标题和关键数据 |
| 次文字 | `#CCCCCC` | 按钮和正文辅助文字 |
| 说明文字 | `#8B8FA8` | caption、label、placeholder |
| P0 风险色 | `#FF4D4D` | 最高优先级 |
| P1 警示色 | `#FF8C00` | 中高优先级 |
| P2 提示色 | `#F5C518` | 中优先级 |

### 字体层级建议

| 层级 | 建议字号 | 字重 | 用途 |
| --- | --- | --- | --- |
| H1 | 20px | 700 | 侧边栏主标题、页面一级标题 |
| H2 | 16-18px | 600 | 模块标题、tab 内主标题 |
| H3 | 14-16px | 600 | 卡片标题、分区标题 |
| Body | 14px | 400-500 | 正文、描述、按钮文字 |
| Caption | 12px | 400 | 辅助说明、表头、label |
| Metric Value | 28px | 700 | 指标数值 |
| Tag | 12px | 600 | P0 / P1 / P2 标签 |

### 设计补充建议

- 按钮统一圆角 8px，卡片统一圆角 12px
- 指标卡统一使用深色容器 + 品牌色底边
- 洞察卡片统一使用左侧优先级色条来强化层级感
- Deep Dive 状态面板建议强化步骤编号和完成态反馈
- 数据表格建议保持高信息密度，但通过行 hover 提升可读性
